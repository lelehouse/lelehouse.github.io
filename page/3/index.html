H5C3--CSS3 介绍

CSS3 介绍

CSS3的新特性

  • 更强大的选择器
  • 更丰富的样式
  • 媲美jquery的动画效果（很多动画不再需要使用js来实现，css就可以搞定了）
  • 适应不同屏幕大小的弹性布局

伪类选择器（befor， after）

  • 所谓伪，就是在html中没有写，在css中却直接用了，在开发工具中也看不到
  • :before 永远添加在选中的元素最前面，有且只有一个
  • :after 永远添加在在选中元素的最后面，有且只有一个
  • before和after必须设置content属性，不然无法显示
  • before和after需要手动指定其display属性

伪类选择器解决浮动的问题

  • 已经学过的清除浮动的两种方式
    • 方式一：添加一个clear的子元素，设置clear: both;
    • 方式二: 设置父元素的overflow：auto;
  • 例用::after清除浮动

响应式开发轮播图的一个优化方案

##移动端响应式开发关于轮播图的一个优化方案
需求是pc端和移动端准备了两套不同尺寸的图片，在不同的终端，显示不同的图片。
常见的解决方案就是：

###第一种：直接在item放置两个标签，通过bootstrap的hidden-sm来控制显示响应大图还是小图，缺点就是不管pc端还是移动端都会请求两个图片，增加了请求和降低的速度。
解决方式用js控制：—》

###优化思路一:$(window).on(“resize”,function(){
}).trigger(“resize”);

该想法就是将图片的数据存放在数组里，通过注册resize事件，在不同终端显示不同的图片，这个做法确实减少了请求，
但是问题是后期维护的时候，图片可能要经常的变动，这样后期还要对数据进行维护，存在不方便的地方。

###针对优化一，提出优化思路二：将两种尺寸的图片名放入item的标签上，用两个自定属性data-pc-img,data-mobile-img存放图片的数据，
在js中就使用data(“”)，获取对应的尺寸的图片，这样既减少了请求的次数，也降低了维护的复杂度。
