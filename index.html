    于是想修改下，按命令 npm config set prefix 结果就遇坑了，之后我的npm命令完全无反应，不是加载的那种状态

    强调：不是nodejs安装目录npm模块下的那个npmrc文件
    而是在C:\Users{账户}\下的.npmrc文件….

    ##JSONP的跨域原理

实践：a.ajax中将dataType设置为”jsonp”
jsonp实现跨域的原理：
1-$.ajax 解决跨域时 将dataType设置为 jsonp即可跨域 为哈？

重要前提：ajax 不能进行跨域请求，浏览器限制的

在跨域的情况下，XMLHttpRequest对象 是不能进行跨域请求的

在跨域的情况下（将dataType设置为 jsonp 时），$.ajax方法并没有使用XMLHttpRequest对象 请求服务器

而是使用了其他的方式来代替的；

如何实现：

原理：

jsonp本质上是利用了script标签src属性具备跨域的能力；

1-客户端要定义好一个函数，通过script标签将函数名传递后台

2-后台获取函数名，将数据 以函数参数的形式 拼接到函数中

3-拼接完成后，后台返回 字符串形式的函数调用

4-浏览器接受到函数调用后，立即执行函数 即可实现功能

需要前后台互相配合完成；

响应式开发轮播图的一个优化方案

##移动端响应式开发关于轮播图的一个优化方案
需求是pc端和移动端准备了两套不同尺寸的图片，在不同的终端，显示不同的图片。
常见的解决方案就是：

###第一种：直接在item放置两个标签，通过bootstrap的hidden-sm来控制显示响应大图还是小图，缺点就是不管pc端还是移动端都会请求两个图片，增加了请求和降低的速度。
解决方式用js控制：—》

###优化思路一:$(window).on(“resize”,function(){
}).trigger(“resize”);

该想法就是将图片的数据存放在数组里，通过注册resize事件，在不同终端显示不同的图片，这个做法确实减少了请求，
但是问题是后期维护的时候，图片可能要经常的变动，这样后期还要对数据进行维护，存在不方便的地方。

###针对优化一，提出优化思路二：将两种尺寸的图片名放入item的标签上，用两个自定属性data-pc-img,data-mobile-img存放图片的数据，
在js中就使用data(“”)，获取对应的尺寸的图片，这样既减少了请求的次数，也降低了维护的复杂度。
